## 二、关键口径（必须一致）

- **视角与相机**：第三人称跟随（Third-Person Follow）。
- **UI 完整性**：一级 UI-01～UI-09、二级 UI-10～UI-16、全局 COMP-01～COMP-02；UI 效果图 18 张，1080×1920 竖屏。
- **底部导航栏（COMP-01）**：首版 4 入口（主页 / 背包 / 商城 / 设置）；「角色属性/技能」由主界面按钮进入；「退出」在设置或确认弹窗。
- **顶部状态栏（COMP-02）**：首版必须头像、等级、金币；预留钻石、体力（可显示 0 或隐藏）。

## 三、首版（MVP）范围与占位字段

### 3.1 首版必须闭环

- **核心流程**：主界面 → 选关 → 进入战斗 → 胜/败结算 → 回到主界面。
- **战斗操控**：虚拟摇杆移动 + 自动普攻 + 2 个主动技能按钮（见 3.4）。
- **经济闭环**：金币获取（战斗/掉落）→ 商城/升级消耗（至少一条可用）。
- **装备闭环**：掉落 → 背包展示 → 穿戴/替换 → 属性变化可见。
- **UI 可达性**：18 个界面/组件可达（允许部分按钮为「预留提示」，但必须不崩溃且能返回）。

### 3.2 占位字段（首版不要求闭环）

| 字段/系统 | 状态 | 首版规则 |
|-----------|------|----------|
| 钻石 Diamond | 预留 | 可显示为 0 或隐藏，不做获取/消耗闭环 |
| 体力 Energy | 预留 | 可显示为 0 或隐藏，不做恢复/消耗闭环 |
| 装备耐久 Durability | 预留 | UI 显示「—」或隐藏，不做损耗/修理闭环 |

### 3.3 技能按钮与技能 ID

- **首版**：**2 个技能**（与 UI 效果图一致）。建议：skillId = 0 → `SK001` 多重箭；skillId = 1 → `SK002` 穿透箭。技能 3 可在 UI 占位（Disabled），不进入首版验收。

### 3.4 验收建议（一句话）

- 首版验收以「可达性 + 核心闭环」优先：能打通主流程，且预留字段不造成误导性强制实现或崩溃/卡死。

## 四、术语与 ID 字典（落地对接锚点）

- **需求/交付编号**：UI-01～UI-16、COMP-01～COMP-02。

### 4.2 技能 ID 体系（首版唯一标准）

- **类型**：`skill_id` 为 **string**，格式 `SK` + 3 位数字（如 `SK001`）。
- **首版技能**：SK001 多重箭、SK002 穿透箭。UI 按钮顺序与 `skill_id` 绑定以配置为准。

**技能配置字段最小集合（推荐 JSON 键名）**：`skillId`、`skillName`、`cooldown`、`damageMultiplier`、`aoeShape`、`range`、`angle`、`arrowCount`。

### 4.3 音频 Key（首版）

- **唯一来源**：`交付文档/音效最小规范与清单_v3.2.md`。任意播放音效的 key 须来自该清单；缺资源时允许占位/复用，但必须不崩溃。
