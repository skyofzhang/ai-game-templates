## 文档依赖关系（AI 理解）

- **规范根**：`00_项目基础规范.md` —— 游戏类型、屏幕、Unity、平台、性能、18 个 UI 清单的唯一口径；四个知识库的「0 章」均引用此文件，不再重复。
- **口径根**：`交付文档/验收与口径.md` —— 版本自包含、口径优先级、首版范围与占位、术语与 ID 字典、验收建议；策划案与交付说明均引用此文件。
- **知识库**：需求 → 策划、程序、美术 均依赖需求；策划、美术 被程序与策划案依赖；所有知识库在「0 章」上引用 00。
- **策划案**：主策划案、UI 策划案、数值策划案 均依据四个知识库 + 00 + 验收与口径；主案不重复 UI/数值细节，引用 UI 策划案与数值策划案。

## 自检清单（补全到无需再补）

- [x] **依据明确**：每个知识库、策划案、交付文档均标明依据（00、验收与口径或其它知识库/策划案）。
- [x] **无待补充**：全包内无「待补充」「TBD」「TODO」等未闭环表述；所有引用目标文件均存在。
- [x] **无断链**：规范与口径仅出自 00 与验收与口径；其它文档仅引用，不重复定义同一口径。
- [x] **引用闭环**：00 ↔ 验收与口径 已做交叉说明；知识库/策划案/交付文档 与 00、验收与口径 的引用关系已补全。

满足上述四项即视为**无需再补**；后续仅做版本迭代（如知识库 v2.3、策划案 v3.3）时更新版本号与更新记录即可。
