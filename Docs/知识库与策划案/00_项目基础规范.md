## 1. 游戏类型

- **游戏类型**：3D Action RPG（3D 动作角色扮演游戏）
- **视角类型**：第三人称跟随（Third-Person Follow）
- **操作方式**：虚拟摇杆 + 技能按钮

---

## 2. 屏幕方向与分辨率

- **屏幕方向**：**竖屏（Portrait）**
- **目标分辨率**：**1080×1920**（宽×高）
- **安全区域**：上下各预留 100 像素，避免刘海屏和底部手势条遮挡
- **UI 布局约束**：所有 UI 必须基于竖屏 1080×1920 设计，禁止横屏布局

---

## 3. Unity 版本与工程约束

- **Unity 版本**：**2022.3.47f1**（LTS）
- **渲染管线**：Universal Render Pipeline (URP)
- **脚本后端**：IL2CPP
- **API 兼容性**：.NET Standard 2.1
- **Player Settings**：iOS 最低 12.0；Android 最低 API 24 (Android 7.0)

---

## 4. 目标平台

- **主要平台**：iOS + Android
- **次要平台**：PC（开发与测试）
- **不支持**：WebGL、主机平台

---

## 5. 性能要求

- **目标帧率**：60 FPS
- **最低设备**：iPhone 8 / Android 8.0
- **同屏怪物上限**：15 只
- **地图尺寸上限**：80 米 × 80 米
- **优化约束**（程序落地时须遵守）：
  - 禁止实时阴影（使用烘焙光照）
  - 禁止后处理（Bloom、DOF 等）
  - UI 使用 Canvas Scaler 的 Scale With Screen Size 模式

---

## 6. UI 界面清单（18 个界面/组件）

以下为需求、策划、程序、美术共同遵守的 UI 完整性口径。

### 6.1 一级界面（9 个）

| 编号 | 名称 | 说明 |
|------|------|------|
| UI-01 | 主界面 | 开始游戏、设置入口、全局导航 |
| UI-02 | 关卡选择 | 关卡解锁状态、星级、全局导航 |
| UI-03 | 战斗界面 | 摇杆、技能、血条、经验条、暂停、背包入口 |
| UI-04 | 背包界面 | 装备槽、背包格、出售，全局导航 |
| UI-05 | 结算界面 | 奖励展示，返回主界面 |
| UI-06 | 角色属性 | 属性、等级、装备，全局导航 |
| UI-07 | 技能界面 | 技能列表、升级，全局导航 |
| UI-08 | 商城界面 | 商品、价格、购买，全局导航 |
| UI-09 | 设置界面 | 音量、画质、语言等，全局导航 |

### 6.2 二级界面（7 个）

| 编号 | 名称 | 说明 |
|------|------|------|
| UI-10 | 关卡确认 | 进入关卡前确认 |
| UI-11 | 道具详情 | 道具说明、使用/出售 |
| UI-12 | 出售确认 | 确认出售 |
| UI-13 | 装备详情 | 装备属性、穿戴/卸下 |
| UI-14 | 技能升级确认 | 升级消耗与效果确认 |
| UI-15 | 暂停菜单 | 继续/设置/退出关卡 |
| UI-16 | 失败界面 | 失败提示、重开/返回主界面 |

### 6.3 全局组件（2 个）

| 编号 | 名称 | 说明 |
|------|------|------|
| COMP-01 | 底部导航栏 | 首版 4 入口：主页 / 背包 / 商城 / 设置；角色/技能由主界面按钮进入，退出在设置或确认弹窗 |
| COMP-02 | 顶部状态栏 | 首版必须：头像、等级、金币；预留：钻石、体力（可显示 0 或隐藏） |

### 6.4 UI 完整性要求

- 一级界面 UI-01～UI-09、二级界面 UI-10～UI-16、全局组件 COMP-01～COMP-02 须全部覆盖。
- 每个可点击元素须有对应界面或反馈。

### 6.5 UI 效果图/资源存放说明

- **UI 效果图（Mockup）**：若存在独立「UI 效果图」文件夹，通常位于工程根目录或 `Docs` 同级；具体以项目内 `Docs/README_Unity工程说明.md` 或 `Docs/文件清单.md` 为准。
- **工程内 UI 资源**：贴图、图集等落地资源见 Unity 工程内 `WarcraftReturn_UI`、`WarcraftReturn_UI_Complete_v2.0` 或 `WarcraftReturn/Assets` 下对应目录；路径与命名以程序知识库及策划案为准。
